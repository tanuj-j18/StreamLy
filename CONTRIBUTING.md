# Contributing to StreamLy

First off, thank you for considering contributing to StreamLy! It's people like you that make StreamLy a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our Issues first. If it doesn't exist, feel free to open a new one!

## Fork & create a branch

If this is something you think you can fix, then fork StreamLy and create a branch with a descriptive name.

## Get the test suite running

Make sure to install the dependencies in both `frontend/` and `backend/`. Run the local development servers to ensure your code works without regressions. Check the `README.md` for full local setup instructions.

## Implement your fix or feature

At this point, you're ready to make your changes! Feel free to ask for help; everyone is a beginner at first.

## Make a Pull Request

At this point, you should switch back to your master branch and make sure it's up to date with StreamLy's master branch.
Then, create a Pull Request with a clear title and description of your changes.

Thank you!
